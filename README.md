# library big data — ml система предсказания популярности книг

лабораторная работа по курсу "Проектирование информационных систем"
итмо, 2026

---

## описание

веб-приложение для библиотек которое:

- загружает большие датасеты о книгах (csv, до 500 000+ строк)
- обучает модель random forest для предсказания популярности книги
- визуализирует аналитику фонда через 4 графика
- рекомендует количество экземпляров новой книги для закупки
- ведёт журнал всех событий в реальном времени

связь с тз библиотечной системы:

- статистика популярности книг (раздел 3.1.6)
- количество новых поступлений (раздел 3.1.6)
- формирование отчётов (раздел 3.1.6)

## ход работы

### 1. постановка задачи

в рамках технического задания на автоматизированную систему библиотечного каталога
был определён раздел, требующий аналитической поддержки: формирование отчётов
о популярности книг и планирование закупок новых поступлений (раздел 3.1.6 тз).

была поставлена задача машинного обучения: по характеристикам книги (жанр, рейтинг,
популярность автора, год издания, цена, число страниц, число выдач) предсказать,
будет ли книга популярной (бинарная классификация, поле is_popular).

практическая ценность: библиотекарь вводит параметры новой книги до закупки
и получает рекомендацию по количеству экземпляров.

### 2. выбор данных

было рассмотрено два источника данных:

**синтетический датасет** — генерируется скриптом generate_data.py.
содержит реалистичные зависимости между признаками:
- популярность автора влияет на рейтинг и число выдач
- жанр влияет на базовый спрос (фэнтези и детектив популярнее науки)
- год издания коррелирует с ценой
- число выдач зависит от рейтинга и жанра

поддерживаемый размер: от 10 000 до 500 000 строк.

**реальный датасет kaggle goodreads** — 11 000+ реальных книг
с рейтингами, числом оценок и числом отзывов.
адаптируется скриптом adapt_goodreads.py: нормализация признаков
через перцентили, балансировка классов через медианные пороги.

### 3. выбор алгоритма

был выбран random forest classifier по следующим причинам:

- устойчив к выбросам и не требует нормализации признаков
- хорошо работает на смешанных данных (числовые + категориальные)
- даёт оценку важности признаков (feature importance) из коробки
- легко масштабируется через параметр n_jobs=-1 (все ядра процессора)
- не склонен к переобучению при большом числе деревьев

параметры модели: 100 деревьев, максимальная глубина 10, разбивка 80/20.

### 4. реализация

приложение реализовано на flask. обучение модели вынесено в отдельный
поток (threading) чтобы не блокировать веб-сервер. статус обучения
сохраняется в json-файл и опрашивается фронтендом каждые 1.5 секунды.

этапы обработки данных:
1. загрузка csv через pandas
2. кодирование жанра через labelencoder
3. разбивка на train/test через train_test_split со стратификацией
4. обучение randomforestclassifier
5. расчёт метрик: accuracy, precision, recall, f1, roc-auc
6. генерация 4 графиков через matplotlib и seaborn
7. сохранение модели через joblib

### 5. результаты

**синтетические данные (500 000 строк):**

| метрика   | значение |
|-----------|----------|
| accuracy  | 95.8%    |
| precision | 95.8%    |
| recall    | 95.9%    |
| f1-score  | 95.9%    |
| roc-auc   | 0.9585   |

время обучения: около 12 секунд на стандартном ноутбуке.

**реальные данные kaggle goodreads (11 043 строки):**

| метрика   | значение |
|-----------|----------|
| accuracy  | 97.7%    |
| precision | 98.5%    |
| recall    | 93.2%    |
| f1-score  | 95.7%    |
| roc-auc   | 0.9584   |

время обучения: около 1 секунды.

высокое качество на обоих датасетах подтверждает что модель корректно
улавливает зависимости между признаками и целевой переменной.

### 6. анализ важности признаков

по результатам обучения на синтетическом датасете:

1. число выдач (times_borrowed) — наибольший вклад, прямой индикатор спроса
2. рейтинг (rating) — второй по важности признак
3. популярность автора (author_popularity) — третий
4. жанр (genre) — заметный вклад, фэнтези и детектив популярнее
5. год издания (year_published) — умеренный вклад
6. цена (price) — слабый вклад
7. число страниц (page_count) — наименее важный признак

вывод: для предсказания популярности книги наиболее важны показатели
реального читательского спроса (выдачи, рейтинг), а не формальные
характеристики (цена, объём).

### 7. выводы

- задача ml успешно интегрирована в контекст библиотечной информационной системы
- система работает как на синтетических так и на реальных данных
- интерфейс позволяет библиотекарю без знания ml получить рекомендацию по закупке
- журнал событий обеспечивает прозрачность работы системы
- архитектура допускает подключение новых источников данных без изменения кода модели

## формат csv

| столбец           | описание                        | тип    |
|-------------------|---------------------------------|--------|
| book_id           | идентификатор книги             | int    |
| genre             | жанр                            | string |
| author_popularity | популярность автора (0.0-1.0)   | float  |
| year_published    | год издания                     | int    |
| price             | цена в рублях                   | float  |
| page_count        | количество страниц              | int    |
| times_borrowed    | число выдач                     | int    |
| rating            | рейтинг (1.0-5.0)              | float  |
| is_popular        | метка: популярная (1) / нет (0) | int    |
